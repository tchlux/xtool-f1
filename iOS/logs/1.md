## Summary

This conversation established the initial direction and working baseline for a private iOS app that sends jobs to a personal xTool F1 Laser Engraver.

## User Goals Captured

- Build a new iOS app for personal use with an xTool F1.
- Keep the app focused on a minimal workflow: define laser settings, define a print path, send the job to the machine.
- Preserve the F1's physical safety workflow: the actual `go`/start action must still be confirmed on the machine.
- Support preview/framing mode, where the blue diode traces the outline at very low power.
- Prefer official/local protocols where available.
- If protocol behavior is unclear, observe the current xTool/LightBurn app traffic before removing or changing capabilities.

## Research Performed

- Checked current xTool support documentation and confirmed xTool does not appear to publish a stable public high-level F1 job API.
- Found xTool's network interface documentation:
  - BLE is used for nearby discovery/provisioning.
  - Wi-Fi and USB are used for device communication.
  - USB on F1-class machines is exposed as an RNDIS/USB Ethernet adapter rather than plain serial.
  - LAN discovery and service ports are documented for newer protocol paths.
- Found xTool's 2026 firmware compatibility notes:
  - F1 firmware `40.51.020.04` or newer is in the newer restricted protocol family.
  - Downgrade-compatible F1 firmware listed by xTool is `40.51.015.02`.
  - LightBurn and older mobile/desktop app compatibility can be affected by newer firmware.
- Found xTool's official LightBurn guide for F1 and verified the downloadable LightBurn device configuration.

Confirmed:

- Port `80`: open
- Port `8780`: open

Read-only HTTP checks:

Read-only GRBL-style probes to port `8780`:

- `$I` produced no visible response.

## LightBurn Configuration Findings

Downloaded and inspected xTool's official `xTool-F1-V2.2.lbdev`.

Important values:

- LightBurn device type: `TCP`
- Network port: `8780`
- Work area: `115 x 115 mm`
- Mirrored Y: `true`
- Baud rate in config: `115200`
- Transfer mode: `0`
- Start G-code: `$L`, `G90`, `G0 F240000`
- End G-code: `M116A127B127`, `G90`, `M6`, `$P`
- Blue laser macro: `M114S1`
- IR laser macro: `M114S2`
- Rotary framing macro: `M116A2B-4`
- Rotary frame close macro: `M116A127B127`

Conclusion: this specific F1 firmware still exposes the older LightBurn-compatible TCP path, so the initial MVP can avoid the newer secured `28900` protocol.

- Uses `preview: true` to cap power at 1%.

- Defaults to dry-run G-code output.

- Uses the blue laser.
- Uses preview mode.
- Generates a centered 20 mm square path in the `115 x 115 mm` work area.
